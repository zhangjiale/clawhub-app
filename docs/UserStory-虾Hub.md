# User Story Map：虾Hub

**来源PRD**：PRD-虾Hub多OpenClaw移动端管理客户端 v1.1
**拆分日期**：2026-06-07
**团队构成**：全栈移动端开发者（1人）
**估算体系**：Fibonacci（1/2/3/5/8/13）
**拆分粒度**：Sprint级（1-3天/Story）

---

## 需求全貌

**涉及角色**：个人效率爱好者（终端用户，唯一角色）

**核心流程**：添加OpenClaw实例 → 浏览Agent列表 → 选择虾进入对话 → 发送/接收消息 → 在消息页查看最近对话 → 查看虾成长数据

**数据实体**：Instance（实例配置）、Agent（虾信息）、Message（对话消息）、Session（会话状态）、UserPreference（用户偏好设置）

**拆分模式选择**：

| PRD功能模块 | 拆分模式 | 理由 |
|------------|---------|------|
| 3.1 实例连接管理 | 按工作流步骤 | 添加实例是线性流程（扫码/手动 → 测试 → 保存） |
| 3.2 Agent列表 | 按工作流步骤 + CRUD | 列表展示+管理操作 |
| 3.3 对话聊天界面 | 按工作流步骤 | 发消息→收回复→展示工具调用 |
| 3.4 消息页 | 按工作流步骤 | 列表渲染+导航 |
| 3.5-3.9 | 按复杂度递进 | 从基础版到完整版渐进交付 |

## Epic 1：MVP — 核心对话能力（第1-4周）

### US-001：扫码添加OpenClaw实例

**角色**：作为养虾用户
**需求**：我希望通过扫描OpenClaw实例生成的二维码来快速添加实例
**价值**：以便不用手动输入复杂的Gateway地址和Token
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 用户点击"添加实例"并选择"扫码添加", When 将摄像头对准OpenClaw配置二维码, Then 自动解析出Gateway URL和Token并填入表单
- [ ] Given 二维码解析成功, When 用户确认实例名称并点击"连接测试", Then 10秒内完成连通性测试，成功则标记"在线"并添加到实例列表
- [ ] Given 二维码格式不合法（非OpenClaw配置码）, When 扫描完成, Then 提示"无法识别的二维码，请确认这是OpenClaw配置二维码"

**依赖**：无
**技术备注**：需调用设备摄像头权限；二维码内容格式需与OpenClaw侧约定JSON Schema

---

### US-002：手动添加OpenClaw实例

**角色**：作为养虾用户
**需求**：我希望通过手动输入Gateway地址和Token来添加实例
**价值**：以便在无法扫码的场景（如远程服务器）也能添加实例
**优先级**：P0
**Story Points**：2

**Acceptance Criteria**：
- [x] Given 用户选择"手动添加", When 填写实例名称、Gateway URL（ws://或wss://开头+端口）、Token并点击"连接测试", Then 执行连通性测试，成功则添加到列表
- [x] Given URL格式不正确（无协议/无端口）, When 用户输入时, Then 输入框实时标红并提示"请输入正确的地址格式，如 ws://192.168.1.100:18789"
- [x] Given Token认证失败, When 连接测试执行, Then 提示"认证失败，请检查Token是否正确"

**依赖**：无（可与US-001并行开发）
**技术备注**：URL校验用正则；Token加密存储到Keychain/Keystore

---

### US-003：查看和编辑实例列表

**角色**：作为养虾用户
**需求**：我希望看到所有已添加实例的状态列表，并能编辑和删除实例
**价值**：以便管理我的多个OpenClaw部署环境
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [x] Given 用户进入"实例"Tab, When 页面加载, Then 展示所有实例卡片，每张卡片包含图标、名称、URL、在线状态和Agent数量
- [x] Given 用户点击实例的"测试连接"按钮, When 测试完成, Then 更新该实例的在线状态显示
- [x] Given 用户滑动删除一个实例并确认, When 删除操作完成, Then 实例从列表消失，该实例下所有本地对话记录同步清除
- [x] Given 应用启动, When 初始化完成, Then 对所有已保存实例执行静默连通性检测并更新状态

**依赖**：US-001或US-002（至少有一种方式添加实例后才有数据可展示，但UI可用Mock数据开发）
**技术备注**：删除操作需二次确认弹窗；启动时检测应异步执行不阻塞UI

### US-004：Agent分组列表展示

**角色**：作为养虾用户
**需求**：我希望在"虾列表"Tab中看到所有实例下的Agent，按实例分组展示
**价值**：以便快速了解我养了哪些虾以及它们分属哪个实例
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 用户打开"虾列表"Tab, When 数据加载完成, Then 按实例分组展示Agent卡片，每个分组头显示实例名称和在线状态点，分组可折叠/展开
- [ ] Given 每个Agent卡片, When 渲染完成, Then 展示虾头像（自动生成）、名称、描述、在线状态指示灯、最后活跃时间
- [ ] Given 某实例处于离线状态, When 渲染该分组, Then 该分组下所有Agent显示为离线状态，卡片视觉降级（灰色调）
- [ ] Given Agent列表拉取失败（Gateway不可达）, When 打开页面, Then 展示本地缓存的上次数据，顶部提示"无法获取最新列表"

**依赖**：US-003（需实例配置数据）；WebSocket连接管理器基础模块
**技术备注**：Agent数据从Gateway API拉取；本地SQLite缓存；支持虚拟滚动（>50个Agent时）

### US-005：虾列表状态统计栏

**角色**：作为养虾用户
**需求**：我希望在虾列表顶部看到全局统计数据（活跃实例数、在线虾数、总消息数）
**价值**：以便一眼掌握我的虾群整体运行状态
**优先级**：P0
**Story Points**：2

**Acceptance Criteria**：
- [ ] Given 用户打开虾列表Tab, When 页面渲染完成, Then 顶部横向展示3个统计卡片：活跃实例数（在线/总数）、在线虾数（在线/总数）、总消息数
- [ ] Given 用户发送了一条消息并收到回复, When 消息处理完成, Then 总消息数统计实时更新
- [ ] Given 某实例从在线变为离线, When 状态变化被检测到, Then 活跃实例数和在线虾数同步更新

**依赖**：US-004（需Agent列表数据）
**技术备注**：统计数据从本地数据实时计算，不需网络请求；使用响应式数据绑定

### US-006：底部三Tab导航框架

**角色**：作为养虾用户
**需求**：我希望App有清晰的底部导航让我在虾列表、消息和实例管理之间快速切换
**价值**：以便我始终知道自己在哪个模块，并能一键跳转
**优先级**：P0
**Story Points**：2

**Acceptance Criteria**：
- [x] Given App启动完成, When 主界面渲染, Then 底部展示三个Tab：虾列表（默认选中）、消息、实例
- [x] Given 用户点击任一Tab, When 页面切换完成, Then 被点击的Tab图标和文字高亮为主题色，其他Tab为灰色
- [x] Given 用户在对话页面点击返回, When 返回动画完成, Then 回到进入对话前的来源Tab，底部导航高亮对应Tab

**依赖**：无（导航框架可最先开发）
**技术备注**：使用移动端Tab导航组件；每个Tab维护独立的页面栈

### US-007：发送文字消息并接收Agent回复

**角色**：作为养虾用户
**需求**：我希望在对话界面发送文字消息并实时收到虾的回复
**价值**：以便通过移动设备和我的AI虾流畅对话
**优先级**：P0
**Story Points**：8

**Acceptance Criteria**：
- [ ] Given 用户从虾列表或消息页点击一只在线虾, When 对话页加载完成, Then 展示虾头像、名称、在线状态，加载本地历史消息，WebSocket连接已建立
- [ ] Given 用户输入文字并点击发送, When 消息发出, Then 立即展示用户消息气泡（右侧），左侧出现"虾思考中"加载动画（三点跳动）
- [ ] Given Agent处理完成返回回复, When 消息到达客户端, Then 替换加载动画为Agent消息气泡（左侧），支持Markdown渲染（粗体、列表、链接）
- [ ] Given Agent回复包含代码块, When 渲染完成, Then 代码块有语法高亮和独立背景色，可横向滚动
- [ ] Given WebSocket连接已断开, When 用户发送消息, Then 消息标记为"待发送"（灰色时钟图标），顶部横幅"连接已断开，正在重连..."
- [ ] Given Agent超过60秒无回复, When 超时触发, Then 展示提示"虾思考时间较长，可能正在处理复杂任务。继续等待/取消"

**依赖**：US-004（需Agent数据）、WebSocket连接管理器
**技术备注**：WebSocket消息格式遵循OpenClaw Gateway协议；Markdown渲染库选型（如marked/markdown-it）；输入框自动高度调整

---

### US-008：展示Agent工具调用状态

**角色**：作为养虾用户
**需求**：我希望在对话中看到虾正在调用什么工具以及执行进度
**价值**：以便了解虾的工作过程，判断任务是否在进行中
**优先级**：P0
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given Agent在回复过程中调用了工具, When 工具调用事件通过WebSocket到达, Then 在消息流中插入工具调用卡片，展示工具名称和"进行中"状态
- [ ] Given 工具调用完成, When 状态更新事件到达, Then 卡片状态变为"✅ 完成"，展示结果摘要
- [ ] Given 工具调用失败, When 错误事件到达, Then 卡片状态变为"❌ 失败"，展示错误原因

**依赖**：US-007（需对话界面基础）
**技术备注**：工具调用是OpenClaw Agent的核心能力，WebSocket事件流中有专门的tool_call事件类型

**角色**：作为养虾用户
**需求**：我希望通过点击预设的快捷指令标签快速发送常用命令

- [ ] Given 用户点击某个快捷指令标签, When 点击触发, Then 指令文本自动填入输入框并立即发送
- [ ] Given 该虾没有配置快捷指令, When 进入对话, Then 快捷指令区域不展示，输入栏正常显示

**依赖**：US-007（需对话界面）
**技术备注**：MVP阶段快捷指令从Agent配置中读取（预设），用户自定义配置在US-014（个性化）中实现

### US-010：消息页（对话列表）

**角色**：作为养虾用户
**需求**：我希望有一个像微信消息列表一样的页面，按最新对话时间展示所有虾
**价值**：以便快速找到最近活跃的虾并进入对话
**优先级**：P0
**Story Points**：5

**Acceptance Criteria**：
- [x] Given 用户点击底部"消息"Tab, When 页面加载完成, Then 展示所有有过对话记录的虾列表，按最后一条消息时间降序排列
- [x] Given 消息列表每一行, When 渲染完成, Then 展示虾头像（含在线状态）、虾名称、最后消息预览（最多40字，用户发的消息前缀"你:"）、消息时间
- [x] Given 某虾有未读新消息, When 列表渲染, Then 该行尾部展示红色未读角标（数字）
- [x] Given 所有虾均无对话历史, When 进入消息页, Then 展示空状态引导页："还没有和任何虾对话过，去虾列表找一只开始聊天吧"
- [x] Given 每次进入消息页, When 页面显示, Then 自动刷新列表数据和排序

**依赖**：US-004（需Agent数据）、US-007（需对话数据）
**技术备注**：消息预览从本地SQLite最新消息中提取；未读计数需本地维护

## Epic 2：V1.1 — 体验增强（第5-6周）

### US-013：虾个性化配置

**角色**：作为养虾用户
**需求**：我希望为每只虾设置独特的头像、昵称和主题色
**价值**：以便一眼区分不同的虾，增强养虾的个性化体验
**优先级**：P1
**Story Points**：5

**Acceptance Criteria**：
- [ ] Given 用户在对话页点击虾头像区域, When 进入配置页, Then 展示头像编辑区、昵称输入框、12色主题色选择器
- [ ] Given 用户选择一张图片作为虾头像, When 保存, Then 头像在对话页和Agent列表中同步更新
- [ ] Given 用户选择了某个主题色, When 保存, Then 该虾对话页面的导航栏和消息气泡色调更新为选定颜色
- [ ] Given 所有配置仅存储在本地, When 用户清除App数据, Then 所有个性化配置恢复默认

**依赖**：US-004（需Agent列表展示）、US-007（需对话页面）
**技术备注**：图片压缩至合理尺寸；配色数据本地持久化

### US-014：快捷指令自定义管理

**角色**：作为养虾用户
**需求**：我希望为每只虾自定义快捷指令的内容和顺序
**价值**：以便把最常用的命令放在最顺手的位置
**优先级**：P1

**Acceptance Criteria**：

- [ ] Given 用户添加了第11个指令, When 点击添加, Then 禁用"+"按钮并提示"每个虾最多10个快捷指令"
- [ ] Given 用户保存了自定义指令, When 回到对话页, Then 快捷指令区域展示更新后的指令集

### US-015：离线消息暂存与待发送队列

**角色**：作为移动用户
**需求**：我希望断网时发的消息不丢失，网络恢复后自动发送
**价值**：以便在地铁、电梯等弱网环境下也能正常使用
**优先级**：P1
**Story Points**：5

**Acceptance Criteria**：
- [ ] Given WebSocket已断开, When 用户发送消息, Then 消息存入本地待发送队列，UI标记为"待发送"（灰色时钟图标）
- [ ] Given 网络恢复且WebSocket重连成功, When 重连完成, Then 按时间顺序依次发送队列中的消息，状态从"待发送"变为"已发送"
- [ ] Given 待发送队列堆积超过20条, When 用户继续发送, Then 顶部提示"有N条消息等待发送，请检查网络连接"

**依赖**：US-007（需对话和WebSocket基础）
**技术备注**：本地SQLite维护pending_messages表；WebSocket重连使用指数退避策略

---

### US-016：断线重连与消息增量同步

**角色**：作为移动用户
**需求**：我希望断线期间虾产生的回复在网络恢复后自动出现在对话中
**价值**：以便我不会错过任何重要信息
**优先级**：P1
**Story Points**：5

**Acceptance Criteria**：
- [ ] Given WebSocket断线后自动重连成功, When 重连完成, Then 从Gateway拉取断线期间Agent产生的新消息，增量合并到本地对话记录
- [ ] Given 消息合并时存在重复（相同消息ID）, When 合并执行, Then 以Gateway返回的为准，本地不重复展示
- [ ] Given 重连3次仍失败, When 最后一次失败, Then 停止自动重连，展示"无法连接到虾，请检查网络或实例状态。点击重试"

**依赖**：US-015（需待发送队列基础）
**技术备注**：消息去重基于Gateway生成的唯一消息ID；增量拉取需Gateway支持since参数

## Epic 3：V1.2 — 增值功能（第7-9周）

### US-017：全局消息搜索

**角色**：作为养虾用户
**需求**：我希望在所有虾的对话历史中搜索关键词
**价值**：以便快速找到忘记是哪只虾说过的信息
**优先级**：P2
**Story Points**：5

**Acceptance Criteria**：
- [ ] Given 用户在搜索页输入关键词, When 输入停止300ms（防抖）, Then 展示所有匹配消息列表，关键词高亮，每条结果显示虾名称和时间
- [ ] Given 用户点击某条搜索结果, When 跳转完成, Then 进入对应虾的对话页并滚动定位到该消息（上下文展示前后各5条）
- [ ] Given 无匹配结果, When 搜索执行, Then 展示空状态："没有找到包含「xxx」的消息"

**依赖**：US-015/016（需本地消息持久化）
**技术备注**：SQLite全文搜索（FTS5）；搜索结果分页（每页20条）

### US-018：推送通知（任务完成提醒）

**角色**：作为养虾用户
**需求**：我希望虾完成重要任务后收到系统推送通知
**价值**：以便不用反复打开App查看虾的状态
**优先级**：P2
**Story Points**：5

**Acceptance Criteria**：
- [ ] Given 用户已开启通知且Agent产生回复消息, When 消息到达, Then 设备收到系统推送通知，包含虾名称和消息摘要（≤50字）
- [ ] Given 用户点击推送通知, When App唤起, Then 直接跳转到对应虾的对话页面
- [ ] Given 当前处于免打扰时段（如22:00-08:00）, When 通知触发, Then 静默存储，免打扰结束后批量推送汇总通知

**依赖**：US-015/016（需后台WebSocket保活）
**技术备注**：iOS后台保活限制严格，需评估Background Task可行性；Android可用Foreground Service

### US-019：虾成长面板

**角色**：作为养虾发烧友
**需求**：我希望看到每只虾的成长统计数据（对话次数、消息数、活跃天数等）
**价值**：以便直观感受养虾的投入和收获，增强成就感
**优先级**：P2
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 用户进入虾详情页, When 成长面板渲染完成, Then 展示6个统计卡片：对话次数、消息总数、工具调用次数、活跃天数、连续天数、首次对话日期
- [ ] Given 用户与某虾的对话数达到里程碑值（如100次）, When 触发条件满足, Then 对话页面弹出庆祝动画（3秒自动消失）
- [ ] Given App刚安装无历史数据, When 进入成长面板, Then 展示空状态引导："刚开始养虾，快去对话吧！"

**依赖**：US-015/016（需本地对话记录持久化）
**技术备注**：统计数据从本地消息记录聚合计算；动画用Lottie或CSS实现

### US-020：成就系统

**角色**：作为养虾发烧友
**需求**：我希望通过养虾解锁成就徽章
**价值**：以便获得持续养虾的动力和乐趣
**优先级**：P2
**Story Points**：3

**Acceptance Criteria**：
- [ ] Given 用户进入虾详情页的"成就"Tab, When 列表渲染完成, Then 展示所有成就，已解锁的彩色显示+标注"已解锁"，未解锁的灰色显示+标注解锁条件
- [ ] Given 用户达成新成就条件（如累计对话100次）, When 条件触发, Then 解锁该成就并在对话页展示庆祝动画
- [ ] Given 用户清空了某虾的对话缓存, When 数据清除完成, Then 该虾相关统计数据清零，依赖统计数据的成就回退为未解锁

**依赖**：US-019（需成长面板基础数据）
**技术备注**：成就条件硬编码在配置中；解锁状态本地持久化

# Splash Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a cross-platform consistent splash page (Android + iOS + Flutter) with brand exposure, a Flutter-side startup gate with tier-gated error handling, and 800ms minimum display time covering the cold-start window.

**Architecture:** Replace `_ConnectionInitializer` with `StartupGate` (state machine `splash` ↔ `app`); fix the broken platform splash by running `flutter_native_splash:create`; introduce `FatalScreen` widget shared between pre-runApp and post-runApp fatal paths; tier-gate init errors (Notification fatal, Connection soft-fail with in-app banner).

**Tech Stack:** Flutter 3.x, Riverpod 2.6, `flutter_native_splash` 2.4.4, `package_info_plus`, `fake_async`, `mocktail`, `flutter_test`.

**Spec:** `docs/superpowers/specs/2026-07-07-splash-page-design.md` v2 (post board review).

## Global Constraints

- **Iron Law 17**: Domain extraction (e.g., `MinDisplayTimer`) MUST be RED-first — write test, run it (RED), then implement, run it (GREEN).
- **Iron Law 14**: Every new widget needs ≥ 2 tests. `FatalScreen` needs 3 (Retry 防双击 added by board). `StartupGate` needs 5 (tier-gate + 软失败 + precacheImage added by board).
- **Iron Law 1**: `lib/domain/` is unchanged (no domain code touched).
- **Iron Law 4**: No `ValueNotifier` + `addListener` + `setState` bridging; state machine uses `setState` inside State.
- **Iron Law 8**: No empty catch blocks; every catch (e, st) does something.
- **Tier-gate errors (board-mandated)**:
  - `NotificationBootstrap.init()` failure → `FatalScreen` (fatal, hard lock)
  - `ConnectionOrchestrator.initialize()` failure → catch locally, write `connectionInitStateProvider`, app shell mounts (in-app banner via existing `ref.listen` in `ClawHubApp`)
- **First-frame defense**: `precacheImage(AssetImage('docs/design/assets/xiahub-splash-v3.png'), context)` in `StartupGate.initState` postFrameCallback. `frameBuilder` is NOT a first-frame defense (board corrected Architect's claim).
- **Retry 防双击**: `FatalScreen` is a `StatefulWidget` with `_retrying` flag; first click sets it, button becomes disabled.
- **Asset path**: `docs/design/assets/xiahub-splash-v3.png` (V3 splash; V4 icon is independent track). Must be in `flutter.assets:` or `Image.asset` crashes.
- **No new dependencies** beyond `package_info_plus`. `flutter_native_splash` / `mocktail` / `fake_async` already in pubspec.
- **No auto-commit**: Only commit when user explicitly asks (per project memory `no-auto-commit`).
- **Dev environment limitation (per CLAUDE.md)**: Windows, no iOS toolchain, no connected device. iOS `LaunchScreen.storyboard` regeneration runs but visual verification needs manual hardware check.

import 'package:flutter/material.dart';
import 'package:claw_hub/app/theme/tokens.dart';
import 'package:claw_hub/ui_kit/default_error_fallback.dart';

/// 可复用 fatal screen。
///
/// 既被 `main.dart` 的 `showFatal` runApp 路径使用（顶层 runApp，外层需要 MaterialApp），
/// 也被 `StartupGate` 的 inline 路径使用（runApp 之内，无外层 MaterialApp）。
///
/// 自带 `Material` + `Directionality` + `SafeArea` 兜底，让两边行为一致。
///
/// Retry 按钮 `_retrying` 守卫（board 修订）：慢设备上连点两次会让第二个
/// `main()` 在第一个 `ProviderScope` 半 teardown 时撞库（`main.dart:41-44`
/// 的 `onDispose` 只能兜一次）。
class FatalScreen extends StatefulWidget {
  const FatalScreen({
    super.key,
    required this.error,
    required this.stackTrace,
    required this.onRetry,
  });

  final Object error;
  final StackTrace stackTrace;
  final VoidCallback onRetry;

  @override
  State<FatalScreen> createState() => _FatalScreenState();
}

class _FatalScreenState extends State<FatalScreen> {
  bool _retrying = false;

  @override
  Widget build(BuildContext context) {
    return Material(
      type: MaterialType.canvas,
      color: XiaColors.bg,
      child: Directionality(
        textDirection: TextDirection.ltr,
        child: SafeArea(
          child: DefaultErrorFallback(
            error: widget.error,
            stackTrace: widget.stackTrace,
            // 保留按钮在树里（`DefaultErrorFallback` 在 `onRetry == null`
            // 时会整棵移除 FilledButton，与本测试的
            // `find.text('重试', warnIfMissed: false)` 语义不符——后者
            // 对应 Flutter SDK「按钮仍在树里但忽略点击」的语义）。
            // 改用 closure 内部 gate：第一次点击 setState + 触发回调，
            // 后续点击命中 `if (_retrying) return;` 直接吞掉。
            onRetry: () {
              if (_retrying) return;
              setState(() => _retrying = true);
              widget.onRetry();
            },
          ),
        ),
      ),
    );
  }
}
```

## Task 6: Add `notificationBootstrapProvider` for testability

**Files:**
- Modify: `lib/app/notifications/notification_bootstrap.dart` — add `ProviderReader` typedef, change field `_ref: WidgetRef` → `_read: ProviderReader`, change constructor signature, replace 11 `_ref.read(x)` call sites with `_read(x)`
- Modify: `lib/main.dart:139` — `NotificationBootstrap(ref)` → `NotificationBootstrap(ref.read)`
- Modify: `test/app/notifications/notification_bootstrap_test.dart` — replace `_MockWidgetRef` with `_MockProviderReader extends Mock implements ProviderReader`, update stubs
- Modify: `lib/app/di/providers.dart` — add import, add `notificationBootstrapProvider` below `connectionInitStateProvider`

**Why:** The spec's `StartupGate` test (Task 7) needs to override `NotificationBootstrap` to inject a `FakeNotificationBootstrap` that throws on `.init()`. Currently `NotificationBootstrap` is instantiated directly as `NotificationBootstrap(ref)` in `_ConnectionInitializer._initConnections()`. Wrapping it in a provider gives tests an override seam.

**Decision: do this BEFORE the `StartupGate` test (Task 7)**, so the test can use `notificationBootstrapProvider.overrideWith(...)`.

**Type-system prerequisite (board 修订 2026-07-08 — revised after implementer verification):** `WidgetRef` and `Ref` are **sibling abstract classes** in Riverpod 2.6.1 — they share method signatures (`T read<T>(ProviderListenable<T>)`) but have **no inheritance relationship** (`WidgetRef` is `abstract class WidgetRef` in `flutter_riverpod/src/consumer.dart:7`; `Ref` is `abstract class Ref<State>` in `riverpod/src/framework/ref.dart:13`). Widening `NotificationBootstrap._ref` from `WidgetRef` to `Ref` therefore breaks BOTH the production call site (`main.dart:139` passes a `WidgetRef`, which is not a `Ref`) and the 2 test sites. The minimal cross-version-safe fix is to **decouple from the ref type entirely**: accept just the read function via a `typedef ProviderReader = T Function<T>(ProviderListenable<T> provider);`. Both `WidgetRef.read` and `ProviderRef.read` are method-tear-offs assignable to this typedef, so all 3 existing call sites work with one-line changes.

import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:package_info_plus/package_info_plus.dart';

import 'package:claw_hub/app/splash/min_display_timer.dart';
import 'package:claw_hub/app/splash/splash_screen.dart';
import 'package:claw_hub/ui_kit/fatal_screen.dart';

enum StartupPhase { splash, app }

/// 启动闸门：splash ↔ app 状态机。
///
/// 在 runApp 之后、`MaterialApp.router` 之前挂载。
/// - splash 阶段：渲染 `SplashScreen`，等 init 完成 + 800ms
/// - app 阶段：渲染 `child`（= `MaterialApp.router`）
/// - 错误（board 修订 tier-gate）：
///   - Tier 1 (Notification fatal) → `FatalScreen`
///   - Tier 2 (Connection soft-fail) → app shell mount，写 connectionInitStateProvider
class StartupGate extends ConsumerStatefulWidget {
  const StartupGate({required this.child});
  final Widget child;

  @override
  ConsumerState<StartupGate> createState() => StartupGateState();
}

class StartupGateState extends ConsumerState<StartupGate> {
  StartupPhase _phase = StartupPhase.splash;
  String _version = '';
  Object? _initError;
  StackTrace? _initStackTrace;

  @override
  void initState() {
    super.initState();
    // 把 splash 资产 decode 提前到 init 等待期内（board 修订：frameBuilder 不是
    // 首帧防御，必须 precacheImage）。postFrameCallback 拿到合法 BuildContext。
    WidgetsBinding.instance.addPostFrameCallback((_) {
      if (!mounted) return;
      precacheImage(
        const AssetImage('docs/design/assets/xiahub-splash-v3.png'),
        context,
      );
    });
    _runStartup();
  }

  Future<void> _runStartup() async {
    try {
      // 1. 读版本号（与 init 并行启动）
      final pkgFuture = PackageInfo.fromPlatform();

      // 2. 启动 init 任务 + 最短展示计时器
      await Future.wait<Object?>([
        _runInitialization(),
        MinDisplayTimer.wait(const Duration(milliseconds: 800)),
      ]);

      // 3. 都满足 → 切到 app 阶段
      final pkg = await pkgFuture;
      if (!mounted) return;
      setState(() {
        _version = 'v${pkg.version}+${pkg.buildNumber}';
        _phase = StartupPhase.app;
      });
    } catch (e, st) {
      // 仅 NotificationBootstrap.init() 抛错会逃逸到这里
      // （_runInitialization 里 ConnectionOrchestrator 失败被局部吞掉）
      if (mounted) {
        setState(() {
          _initError = e;
          _initStackTrace = st;
        });
      }
    }
  }

  Future<void> _runInitialization() async {
    // Tier 1 (fatal) — 失败时不应进 app shell
    final bootstrap = ref.read(notificationBootstrapProvider);
    await bootstrap.init();

    // Tier 2 (soft-fail) — 失败时记日志 + 写 connectionInitStateProvider
    try {
      final orchestrator = ref.read(connectionOrchestratorProvider);
      await orchestrator.initialize();
    } catch (e, st) {
      ref.read(loggerProvider).error(
        '[startup] orchestrator init soft-failed: $e', st);
      ref.read(connectionInitStateProvider.notifier).state =
          AsyncValue.error(e, st);
    }
  }

  @override
  Widget build(BuildContext context) {
    if (_initError != null) {
      return FatalScreen(
        error: _initError!,
        stackTrace: _initStackTrace!,
        // Retry 重入 _runStartup 而不是调 main()：
        // 1. 避免 startup_gate.dart → main.dart 循环 import
        //    （main.dart 自身就要 import StartupGate）
        // 2. 跳过 bootstrapApp 里的 Workmanager().initialize() 第二次调用
        //    （review #4 风险点：workmanager 0.9.0+3 二次 init 行为不确定）
        onRetry: () {
          setState(() {
            _initError = null;
            _initStackTrace = null;
            _phase = StartupPhase.splash;
            _version = '';
          });
          _runStartup();
        },
      );
    }
    return switch (_phase) {
      StartupPhase.splash => SplashScreen(version: _version),
      StartupPhase.app => widget.child,
    };
  }
}

// Task 8 的 main.dart 需要跨文件引用它。如果保留下划线前缀，
// Dart 编译器会判为 library-private 而 break build（参考 review #1）。
```
